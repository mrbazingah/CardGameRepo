# Multiplayer Architecture - Deterministic Action-Based Sync

## Overview

The multiplayer system uses a **host-authoritative, deterministic, action-based** architecture. This ensures:
- âś… No desyncs (both sides generate same deck from seed)
- âś… Minimal network traffic (only actions, not state every frame)
- âś… Clean separation of concerns
- âś… Predictable and debuggable

---

## Architecture Principles

### 1. Host is Authoritative
- Host generates deck seed
- Host validates all actions
- Host makes all game state decisions

### 2. Deterministic Deck Generation
- Host generates random seed
- Seed is synced to all clients
- Both host and clients generate identical deck order from seed
- Uses `DeterministicDeck.GenerateDeck(seed)` utility

### 3. Action-Based Sync (Not State-Based)
- **Only sync actions**: "Deal cards", "Play card", "Draw card", "Turn advance"
- **Never sync**: Card GameObjects, positions, transforms every frame
- Clients generate visual representation locally from actions

### 4. Minimal Synced State
Only these are networked:
- `DeckSeed` - For deterministic deck generation
- `DeckCount` - Remaining cards in deck
- `PileCount` - Cards in pile
- `PileValues[]` - Card values in pile (via RPC)
- `CurrentTurn` - Whose turn it is
- `GameStarted` - Game state flag
- `HandCount` - For opponent display

## Component Breakdown

### `DeterministicDeck.cs`
**Purpose**: Utility for generating identical decks from a seed.

**Key Methods**:
- `GenerateDeck(int seed, bool removeSpecialCards)` - Creates deck from seed
- `DrawCard(List<byte> deck)` - Removes and returns first card
- `DrawCards(List<byte> deck, int count)` - Draws multiple cards

**Usage**: Both host and client call this with the same seed to get identical deck order.

---

### `NetworkCardGenerator.cs`
**Purpose**: Manages deck and card dealing.

**Synced State**:
- `DeckSeed` - Random seed generated by host
- `DeckCount` - Remaining cards
- `CardsDealt` - Whether initial deal is complete
- `CanDrawChanceCard` - Chance card cooldown

**Local State**:
- `localDeck` - Generated from seed (not synced)

**Actions (RPCs)**:
- `RPC_DealHandCards(PlayerRef, byte[])` - Sends card values to deal
- `RPC_DealSideCards(PlayerRef, byte[], byte[])` - Sends side card values
- `RPC_ReceiveDrawnCards(PlayerRef, byte[])` - Sends drawn card values
- `RPC_ChanceCardResult(PlayerRef, byte)` - Sends chance card value

**Flow**:
1. Host generates seed â†’ `DeckSeed` synced
2. All clients generate deck from seed locally
3. Host deals cards â†’ sends `byte[]` values via RPC
4. All clients create local Card GameObjects from values

---

### `NetworkPlayerHand.cs`
**Purpose**: Manages player's hand (local GameObjects, not NetworkObjects).

**Synced State**:
- `NetworkedHandCount` - For opponent display
- `savedCardValue` - Game logic state
- `hasDiscarded` - Game logic state
- `canEndTurn` - UI state

**Local State**:
- `handCards` - List of local Card GameObjects
- `underSideCards` - List of local Card GameObjects
- `overSideCards` - List of local Card GameObjects

**Actions (RPCs)**:
- `RPC_PlayCard(byte cardValue)` - Player wants to play a card
- `RPC_RemoveCardFromHand(PlayerRef, byte)` - Remove card from hand (all clients)

**Key Methods**:
- `CreateLocalCard(byte value, bool faceUp)` - Creates Card GameObject locally
- `AddHandCardsLocally(byte[] values)` - Called by RPC to add cards
- `SetSideCardsLocally(byte[], byte[])` - Called by RPC to set side cards
- `AddChanceCardLocally(byte value)` - Called by RPC to add chance card

**Flow**:
1. RPC receives card values â†’ `AddHandCardsLocally(byte[])`
2. Creates local Card GameObjects from values
3. Player clicks card â†’ `RPC_PlayCard(byte value)`
4. Host validates and processes
5. All clients remove card locally via `RPC_RemoveCardFromHand`

---

### `NetworkPile.cs`
**Purpose**: Manages central pile of played cards.

**Synced State**:
- `PileCount` - Number of cards in pile
- `IsDiscarding` - Whether pile is being discarded

**Local State**:
- `pileCards` - List of local Card GameObjects
- `pileValues` - Synced via RPC (not Networked property)

**Actions (RPCs)**:
- `RPC_AddCardToPile(byte value, PlayerRef)` - Add card to pile
- `RPC_UpdatePileValues(byte[])` - Sync entire pile state
- `RPC_DiscardPile()` - Discard pile
- `RPC_AddPileCardsToHand(PlayerRef, byte[])` - Pick up pile

**Flow**:
1. Player plays card â†’ `RPC_AddCardToPile(byte value)`
2. Host adds to `pileValues` list
3. Host broadcasts `RPC_UpdatePileValues(byte[])` to all clients
4. All clients regenerate pile cards locally from values

---

### `GameManagerNetwork.cs`
**Purpose**: Manages game state, turns, win conditions.

**Synced State**:
- `CurrentTurn` - Whose turn it is
- `GameStarted` - Whether game has started
- `WinnerDeclared` - Whether game is over
- `PlayersReady` - Ready player count

**Actions (RPCs)**:
- `RPC_RequestNextTurn(byte lastCardValue)` - Advance turn
- `RPC_BroadcastTurn()` - Notify all clients of turn change
- `RPC_BroadcastWin(PlayerRef)` - Notify winner

**Flow**:
- Only essential state is synced
- Actions trigger state changes
- All clients receive state updates

## Data Flow Examples

### Example 1: Dealing Initial Cards

```
1. Host: Generate seed â†’ DeckSeed synced
2. All: Generate deck from seed locally
3. Host: DealInitialHands() â†’ Draws from localDeck
4. Host: RPC_DealHandCards(player, byte[]) â†’ Sends card values
5. All: AddHandCardsLocally(byte[]) â†’ Creates local Card GameObjects
```

### Example 2: Playing a Card

```
1. Client: Player clicks card â†’ RPC_PlayCard(byte value)
2. Host: Validates action â†’ Processes game logic
3. Host: RPC_AddCardToPile(byte value) â†’ Adds to pile
4. Host: RPC_UpdatePileValues(byte[]) â†’ Syncs pile to all clients
5. Host: RPC_RemoveCardFromHand(PlayerRef, byte) â†’ Removes from hand
6. All: Regenerate pile cards locally
7. All: Remove card from hand locally
```

### Example 3: Drawing Cards

```
1. Client: Needs cards â†’ RPC_DrawNewCard(PlayerRef, int amount)
2. Host: Draws from localDeck â†’ Gets byte[] values
3. Host: RPC_ReceiveDrawnCards(PlayerRef, byte[]) â†’ Sends values
4. All: AddHandCardsLocally(byte[]) â†’ Creates local Card GameObjects
```

## Benefits

1. **No Desyncs**: Both sides generate identical deck from seed
2. **Low Bandwidth**: Only actions sent, not state every frame
3. **Predictable**: Deterministic = easy to debug
4. **Scalable**: Can add more players without major changes
5. **Clean Code**: Clear separation between network and local logic
